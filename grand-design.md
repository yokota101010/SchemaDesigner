# DB Architect (Schema Designer) グランドデザイン

本ドキュメントは、本ツール（DB Architect）が単なる「物理DBスキーマ設計ツール」から、**「ドメイン駆動設計（DDD）モデリングをサポートし、生成AIによるシステム開発の『インプット設計図』となるアーキテクチャ設計ツール」**へと進化するための全体像とロードマップを定義したものです。

---

## 1. 背景と目的

最終的に**「システム開発を生成AI（LLM）を軸に行う」**未来を見据えたとき、開発において最も重要な人間の役割は、**「曖昧なビジネス要求を整理し、整合性の取れたビジネスドメインモデルやルール（インプット設計図）を構造化データとして作成すること」**になります。

本ツールは、人間がビジネスドメイン（集約、ユースケース、値オブジェクト、ビジネスルール）を簡単かつクリーンにモデリングでき、それを**「生成AIが解釈して高品質な実装コード（Java等）やインフラ構成を出力できる高品質なJSONデータ（設計図）」**として出力することを目的にします。

試行段階における不要な複雑さを排除するため、厳密なDSL（ドメイン固有言語）などの定義は行わず、**「論理モデリング ＋ 自然言語によるビジネスルールの埋め込み」**というシンプルなアプローチを採用します。

---

## 2. コアコンセプトと設計原則

本ツールの設計における中核となるコンセプトと原則です。

### ① 集約（Aggregate）: トランザクション整合性の境界
*   **概念**: トランザクション整合性を保つための最小の境界（テーブルのグループ）。
*   **役割の区別**:
    *   **集約ルート（Root）**: 集約の外部からのアクセス・トランザクションのエントリーポイントとなるメインテーブル。
    *   **集約メンバー（Member）**: 集約ルートに内包され、ライフサイクルを共にするテーブル（例: 注文に対する注文明細）。
*   **原則**: 外部のオブジェクトは、集約ルート以外のテーブル（Member）を直接参照または変更してはならない。

### ② 機能（ユースケース / バッチ / API / 画面）: トランザクションの実行単位
*   **概念**: ユーザー操作やシステムイベントによって起動される1つの処理単位。
*   **原則**: **「1トランザクション（1機能の実行）で変更（Create/Update/Delete）できるのは、最大1つの集約インスタンスのみ」**とする。
*   **紐付け**: 各「機能」に対して、変更対象となる「集約」を1対1（または1対0）でマッピングする。

### ③ 値オブジェクト（Value Object）: 属性の組み合わせとビジネスルールの最小単位
*   **概念**: 複数のプロパティ（属性）の組み合わせと、それに紐づく「ビジネスルール（自然言語）」を持つオブジェクト。
*   **原則**: 物理的には個別のカラムにフラット化されますが、ドメインモデリング上は1つの型（オブジェクト）として扱います。
*   **例**: `Money`（金額 `amount` と 通貨 `currency`）、`Email`（値 `value`）など。

### ④ 自然言語によるビジネスルールの埋め込み
*   **概念**: カラム単位の「導出項目（derivation）」や「条件」、値オブジェクトの「ルール記述欄」に、**生成AIが解釈可能な自然言語**でルールをそのまま記述します。
*   **メリット**: ツールのメタモデルを複雑にすることなく、あらゆる複雑なビジネスルール（バリデーション、不変条件、計算ロジック）を簡潔に表現できます。

## 3. ヘキサゴナル3ビューアーキテクチャ (ツールの整理)

ツールの複雑化を防ぎ、ターゲット技術（Java等）やインフラの変更に柔軟に対応するため、UIをヘキサゴナルアーキテクチャの3階層に呼応する**3つの明確に分離されたビュー（モジュール）**に分割します。裏側の永続化JSONデータファイルは1つです。

```mermaid
graph TD
    subgraph Domain View (ドメイン層 / 論理モデリング)
        Canvas[論理ER図 Canvas]
        LogicalEditor[テーブル・属性定義]
        ValueObjectEditor[値オブジェクト定義]
        AggregateMap[集約マッピング]
    end

    subgraph Application View (アプリケーション層 / ユースケース)
        UseCaseMap[機能 × 集約マッピング]
        CrudMatrix[CRUDマトリクス]
    end

    subgraph Infrastructure View (インフラストラクチャ層 / 物理マッピング・生成)
        PhysicalMap[物理型・インデックスマッピング / VOの物理フラット化]
        DDLGen[DDL / SQL生成]
        CodeGen[生成AIやテンプレートによるコード・インフラ定義の自動生成]
    end
```
