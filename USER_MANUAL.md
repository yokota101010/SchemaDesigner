# DB Architect (Schema Designer) ユーザーマニュアル

DB Architect は、ブラウザ上で動作するスキーマ設計・データベース設計ツールです。
単なる物理データベースの設計にとどまらず、**ドメイン駆動設計（DDD）**の思想を取り入れた「集約」や「値オブジェクト」のモデリングをサポートし、最終的に生成AI（LLM）に入力してシステムを自動開発するための「インプット設計図（Markdownドキュメント）」を作成することができます。

## 1. 基本コンセプト

DB Architect は、生成AIを軸としたシステム開発の「インプット設計図」を作成することを目的として設計されています。

*   **集約 (Aggregate) の定義**: トランザクション整合性を保つ境界を定義し、データの整合性を担保します。
*   **値オブジェクト (Value Object) の活用**: 複数の関連する属性とビジネスルールを1つの「型」として再利用します。
*   **自然言語ルールの埋め込み**: 導出項目や制約条件にAIが解釈可能な「自然言語」で直接ビジネスルールを記述します。これにより、複雑なバリデーションや計算ロジックを簡潔に表現できます。

## 2. 画面構成とナビゲーション

アプリは直感的に操作できるシングルページアプリケーション (SPA) 構成です。

```
+---------------------------------------------------------------------------------+
|  [Header]  プロジェクト名編集   [新規] [インポート] [エクスポート]  [値オブジェクト] [集約] [AI生成] [🔑] |
+---------------------------------------------------------------------------------+
|  [Canvas]                                                                      |
|  +---------------------------+  +---------------------------+                  |
|  | [メインビュー] (Tab)       |  | [サブビュー] (Tab)         |                  |
|  |                           |  |                           |                  |
|  |  +---------------------+  |  |  +---------------------+  |                  |
|  |  | テーブルA (ドラッグ可)  |  |  | テーブルB (マスター等)   |  |                  |
|  |  +---------------------+  |  |  +---------------------+  |                  |
|  +---------------------------+  +---------------------------+                  |
+---------------------------------------------------------------------------------+
```

### ヘッダー
*   **プロジェクト名**: 左上の入力欄からプロジェクト名を自由に変更できます。
*   **新規・インポート・エクスポート**: プロジェクトのライフサイクル管理を行います。
*   **ドメイン設定ボタン**:
    *   `値オブジェクト`: 値オブジェクト定義モーダルを開きます。
    *   `集約定義`: 集約定義・並び替えモーダルを開きます。
*   **AI生成 & 設定 (🔑)**:
    *   `AIモックデータ生成`: スキーマに合わせたデータの自動生成を行います。
    *   `🔑 (鍵アイコン)`: Gemini APIのAPIキーを設定・保存します。

### キャンバス
*   **ビューの切り替え (メインビュー / サブビュー)**: 
    キャンバス上部のタブでビューを切り替えられます。トランザクションデータなどを「メインビュー」、マスタテーブルなどを「サブビュー」に分けることで、大規模なスキーマでも整理して設計できます。
*   **ドラッグ＆ドロップ**: テーブルのヘッダー部分をドラッグすることで自由に配置を変更できます。
*   **パン (スクロール) 操作**: テーブル以外の背景部分をドラッグすることで、キャンバス全体をスクロールできます。
*   **テーブル間のリレーション**: テーブル定義で設定された外部キー (FK) に基づき、テーブル同士が動的なラインで結ばれます。

## 3. テーブルとカラムの基本操作

### テーブルの操作
1.  **テーブルの追加**: ヘッダーの「テーブル追加」ボタンをクリックすると、キャンバス中央に新しいテーブルが作成されます。
2.  **テーブルの編集**: 各テーブルの右上にある「⚙️ (ギアアイコン)」または「編集」をクリックすると、詳細エディタが開きます。
3.  **テーブルの最小化**: テーブルの左上にある「▲/▼」ボタンで、カラム表示を最小化できます。不要なテーブルを折りたたむのに便利です。
4.  **行データ（レコード）の追加・削除**: キャンバス上で直接テーブルに簡易的なデータを追加・編集できます。

### カラムの編集 (テーブルエディタ内)
テーブルエディタ（「⚙️」をクリックして開くモーダル）では、以下の項目を設定できます。

*   **PK (主キー)**: 鍵マークをクリックして主キーを設定します。
*   **カラム名**: 物理的なカラム名を英名で指定します。
*   **データ型**: 基本データ型 (`INT`, `VARCHAR(255)`, `DATE` 等) のほか、定義済みの**「値オブジェクト」**や、サブビューにあるテーブルへの**「マスタ参照 (FK)」**を選択できます。
*   **区分 (独立 / 導出項目)**:
    *   `独立 (independent)`: 生のデータ項目。
    *   `導出項目 (dependent)`: 他の項目から計算・導出される項目。
*   **説明**: カラムの役割や、AIがダミーデータを生成する際のルール（例: `有効/無効`、`1〜50の範囲`）を記述します。右端の編集アイコンから、より詳細な説明・指示をポップアップで記述できます。
*   **詳細設定 (導出式)**: 区分を「導出項目」に設定した場合に表示されます。計算ロジックや参照元を自然言語で記述します（例: `Table.Col via FK`）。
*   **表示スイッチ (目玉マーク)**: カラムの画面上での表示/非表示を切り替えます（キー項目は常に表示されます）。

### 制約の設定
*   **外部キー (FK) 制約**:
    エディタ右上にある「FKを追加」をクリックすると、テーブルにFK列（動的カラム）が追加されます。
    1.  追加されたFK列のヘッダーから「親テーブル」を選択します。
    2.  各カラムの交点にあるチェックボックスをONにして、外部キーに含めるカラムを指定します。
    3.  チェックをONにした行のドロップダウンから「対応する親テーブルのカラム」をマッピングします。
*   **ユニークキー (UQ) 制約**:
    エディタ右上の「UQを追加」をクリックすると、UQ列が追加されます。一意制約に含めたいカラムのチェックボックスをONにします（複数選択による複合ユニークキーにも対応）。

## 4. ドメインモデリング機能 (DDDサポート)

### 値オブジェクト (Value Object)
値オブジェクトは、複数のプロパティとビジネスルールをセットにした「ドメイン型」です。

1.  ヘッダーの「値オブジェクト」ボタンをクリックします。
2.  「追加」ボタンから新しい値オブジェクトを作成します。
3.  **ビジネスルール・制約条件**: 生成AIがデータ整合性を理解するためのビジネスルールを記述します（例: `金額(amount)は0以上の正の数。ただし通貨が'JPY'の場合は整数のみ`）。
4.  **プロパティ**: 値オブジェクトを構成する要素（英名、物理データ型、説明）を定義します。
5.  **テーブルへのマッピング**:
    *   テーブルエディタのカラムの「データ型」ドロップダウンから、作成した値オブジェクト名を選択します。
    *   テーブル内に `└─ (プロパティ名)` という形で、値オブジェクトのプロパティが**自動的にフラット展開**されます。
    *   展開された内部プロパティは、値オブジェクト側で一括管理されます。

### 集約 (Aggregate)
集約は、トランザクション整合性を保つ境界（テーブルのグループ）です。

1.  ヘッダーの「集約」ボタンをクリックして集約定義を開きます。
2.  「集約マスター管理」で集約（グループ名）を追加・変更します。
3.  **テーブルの割り当て**:
    各テーブルがどの集約に属するかを選択し、役割を設定します。
    *   `R (Root)`: 集約の外部からのアクセス・トランザクションのエントリーポイントとなるメインテーブル。
    *   `M (Member)`: 集約ルートに内包され、ライフサイクルを共にする子テーブル。
4.  **🧹 集約ごとに整列**:
    「集約ごとに整列」ボタンをクリックすると、割り当てられた集約ごとにテーブルリストが一括でソートされます。

## 5. AIを活用したテスト用モックデータ自動生成

Gemini APIを利用して、設計したスキーマ、リレーションシップ、および値オブジェクトのビジネスルールに基づいた、リアルな日本語テストデータを一括生成できます。

### 事前準備
1.  ヘッダーの「🔑 (鍵アイコン)」をクリックし、お手持ちの **Gemini APIキー** を入力して保存します。
    *(※入力したAPIキーはブラウザのLocalStorageにのみ安全に保管され、外部のサーバー等に送信されることはありません)*

### データ生成手順
1.  ヘッダーの「AI生成」をクリックします。
2.  **生成プロンプトの指定**:
    *   `初期レコードの指示`: データの起点となる初期状態の指示（例: `ユーザーは3人作成、それぞれに紐づく家計簿レコードを作成`）を入力します。
    *   `その他指示（任意）`: AIに考慮してほしい特別なビジネスルール（例: `日付は2026年6月の範囲に収めてください`）を入力します。
3.  「生成」をクリックすると、AIによるデータ生成プロセスが開始されます。

### AI生成の高度な制御仕様
*   **参照整合性の厳守**: 外部キー (FK) のマッピングをAIが解釈し、親テーブルに存在しない値が子テーブルに設定されないよう、依存関係順に整合性を持って生成します。
*   **二段階の一括生成パイプライン**:
    *   **Phase 1**: 全テーブルの独立カラム（生のデータ）を一括生成。
    *   **Phase 2**: 1段階目の生成結果をもとに、導出カラム（計算項目や依存値）の計算結果を一括で算出（Chain of Thought による自己検算を実施して計算の正確性を担保）。
*   **エラーハンドリングとレートリミット対策**:
    Gemini APIのレートリミット (HTTP 429) 回避のため、リクエスト間に自動で5000msのウェイトを挿入し、失敗時は指数バックオフ（最大7回自動リトライ）を実行します。

## 6. データの保存とインポート・エクスポート

*   **自動保存 (LocalStorage)**:
    設計内容は、操作するたびにブラウザのローカルストレージへリアルタイムに自動保存されます。ブラウザをリロードしたり閉じたりしても、次回の起動時に自動で前回の状態が復元されます。
*   **Markdownエクスポート (保存)**:
    ヘッダーの「仕様書保存 (.md)」をクリックするか、ショートカットキー **`Ctrl + S` (macOSでは `Cmd + S`)** を押すことで、現在のプロジェクト全体の設計情報・データを記載したMarkdownファイル（`spec.md`）をダウンロードできます。
*   **Markdownインポート (開く)**:
    ヘッダーの「仕様書読込 (.md)」からエクスポートしたMarkdownファイルを読み込むことで、過去の設計状態を完全に復元できます。

## 7. オフライン利用と配布

本ツールはビルド時に**単一のHTMLファイル (`dist/index.html`)** にコンパイルされます。

*   **配布の容易さ**: 生成された `dist/index.html` 1ファイルだけをUSBで持ち運んだり、メール等で共有したりできます。
*   **オフライン動作**: 依存アセット (JS, CSS) がすべてインライン化されているため、インターネット接続がない環境（閉域網やネットワーク制限のある社内環境など）でも、ダブルクリックするだけで完全に動作します。
    *(※AI生成機能を使用する場合のみ、Gemini APIへのインターネット通信が必要となります)*
